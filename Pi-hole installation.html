Pi-hole Server Installation & Management

Pi-hole is a network-wide ad blocker that acts as a DNS sinkhole, protecting your devices from unwanted content without requiring any client-side configuration.

Prerequisites

  • Ubuntu 20.04/22.04 LTS or RHEL 8/Rocky Linux 8/AlmaLinux 8
  • Minimum 1GB RAM (2GB recommended)
  • At least 2GB of free disk space
  • Static IP address configured
  • Root or sudo privileges
  • Basic knowledge of networking and DNS
Ubuntu
RHEL/CentOS/Rocky

1 System Preparation (Ubuntu)

Update your system and install required dependencies:

sudo apt update && sudo apt upgrade -y
sudo apt install curl git sqlite3 dnsutils -y

Set a static IP address (if not already configured):

# Edit network configuration
sudo nano /etc/netplan/01-netcfg.yaml

# Example configuration:
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses: [192.168.1.10/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]

Apply the network configuration:

sudo netplan apply

2 Install Pi-hole (Ubuntu)

Download and install Pi-hole using the automated installer:

curl -sSL https://install.pi-hole.net | bash
Warning: Always review scripts before piping them to bash. You can examine the script first with: curl -sSL https://install.pi-hole.net

Follow the interactive installer prompts:

  1. Press OK to continue through the welcome screens
  2. Select your upstream DNS provider (Google, Cloudflare, OpenDNS, etc.)
  3. Choose the network interface to listen on (usually eth0)
  4. Select protocols (IPv4 and/or IPv6)
  5. Choose to install the web admin interface
  6. Install the web server (lighttpd)
  7. Enable query logging (recommended)
  8. Set a privacy level for the logs
  9. Set a secure admin password when prompted

1 System Preparation (RHEL/Rocky/AlmaLinux)

Update your system and install required dependencies:

sudo dnf update -y
sudo dnf install epel-release -y
sudo dnf install curl git sqlite3 bind-utils -y

Set a static IP address (if not already configured):

# Edit network configuration
sudo nmcli connection modify "Wired connection 1" ipv4.method manual ipv4.addresses "192.168.1.10/24" ipv4.gateway "192.168.1.1" ipv4.dns "8.8.8.8,1.1.1.1"
sudo nmcli connection up "Wired connection 1"

Disable or configure firewalld:

# Option 1: Disable firewalld (not recommended)
sudo systemctl stop firewalld
sudo systemctl disable firewalld

# Option 2: Configure firewalld (recommended)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --permanent --add-service=dns
sudo firewall-cmd --reload

2 Install Pi-hole (RHEL/Rocky/AlmaLinux)

Download and install Pi-hole using the automated installer:

curl -sSL https://install.pi-hole.net | bash
Warning: Always review scripts before piping them to bash. You can examine the script first with: curl -sSL https://install.pi-hole.net

Follow the interactive installer prompts (same as Ubuntu installation).

If you encounter SELinux issues, you may need to adjust policies:

# Check SELinux status
sudo sestatus

# If enforcing, consider setting to permissive
sudo setenforce 0
sudo sed -i 's/^SELINUX=.*/SELINUX=permissive/' /etc/selinux/config

3 Post-Installation Configuration

After installation, note the admin password displayed at the end, or set a new one:

pihole -a -p

Access the web admin interface:

http://your_server_ip/admin

Configure your router to use Pi-hole as the DNS server, or manually configure devices:

# Manual device configuration:
# Set DNS server to your Pi-hole IP address (e.g., 192.168.1.10)

4 Securing Pi-hole

Change the default admin password:

pihole -a -p

Enable HTTPS for the admin interface:

# Install certbot and get SSL certificate
sudo apt install certbot -y  # Ubuntu
sudo dnf install certbot -y  # RHEL/Rocky

# Obtain certificate
sudo certbot certonly --webroot -w /var/www/html -d pihole.yourdomain.com

Configure lighttpd to use SSL:

sudo nano /etc/lighttpd/lighttpd.conf

# Add SSL configuration
$SERVER["socket"] == ":443" {
  ssl.engine = "enable"
  ssl.pemfile = "/etc/letsencrypt/live/pihole.yourdomain.com/fullchain.pem"
  ssl.privkey = "/etc/letsencrypt/live/pihole.yourdomain.com/privkey.pem"
  server.name = "pihole.yourdomain.com"
  server.document-root = "/var/www/html"
}

Restart lighttpd:

sudo systemctl restart lighttpd

Set up fail2ban to protect against brute force attacks:

sudo apt install fail2ban -y  # Ubuntu
sudo dnf install fail2ban -y  # RHEL/Rocky

# Create Pi-hole jail configuration
sudo nano /etc/fail2ban/jail.d/pihole-dns.conf

[pihole-dns]
enabled = true
port = 53
protocol = udp
filter = pihole-dns
logpath = /var/log/pihole.log
maxretry = 3
bantime = 3600

5 Managing Devices and Access Control

Add devices to the network and configure Pi-hole to manage them:

a) Group Management

Create groups for different device types:

# Via web interface: Groups -> Manage Groups
# Or via CLI:
sqlite3 /etc/pihole/gravity.db "INSERT INTO 'group' (name, description) VALUES ('kids_devices', 'Children\\'s devices');"

b) Client Management

Add clients manually or via DHCP:

# Add a client manually
pihole -a staticdhcp   

# Example:
pihole -a staticdhcp AA:BB:CC:DD:EE:FF 192.168.1.50 my-laptop

c) Access Control

Control which devices can use Pi-hole:

# Permit only specific IP ranges
sudo nano /etc/pihole/pihole-FTL.conf

# Add these lines:
BLOCKINGMODE=NXDOMAIN
BLOCK_ICMP=always
API_QUERY_LOG_SHOW=all
RATE_LIMIT=1000/60

6 Maintenance and Monitoring

a) Regular Updates

Keep Pi-hole and your system updated:

# Update Pi-hole
pihole -up

# Update system packages
sudo apt update && sudo apt upgrade -y  # Ubuntu
sudo dnf update -y  # RHEL/Rocky

b) Backup and Restore

Backup your Pi-hole configuration:

# Create backup
pihole -a -t

# Schedule automatic backups
echo "0 2 * * 0 pihole -a -t" | sudo tee -a /etc/crontab

c) Monitoring and Logs

Monitor Pi-hole performance and logs:

# Check service status
pihole status

# View real-time query log
pihole tail

# Check memory usage
pihole memory

# Get statistics
pihole -c

7 Advanced Configuration

a) Custom Block Lists

Add additional blocklists for enhanced protection:

# Via web interface: Group Management -> Adlists
# Or via CLI:
sqlite3 /etc/pihole/gravity.db "INSERT INTO adlist (address, enabled, comment) VALUES ('https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts', 1, 'StevenBlack Unified Hosts');"

Update gravity database after adding new lists:

pihole -g

b) Custom DNS Records

Add local DNS records for your network:

# Add custom DNS record
pihole -a hostrecord router.local 192.168.1.1

# Add CNAME record
pihole -a hostrecord nas.local 192.168.1.100

c) DHCP Server Configuration

Enable Pi-hole's built-in DHCP server:

# Enable DHCP server
pihole -a enabledhcp

# Configure DHCP range
pihole -a setdhcp 192.168.1.100 192.168.1.200 255.255.255.0 24h

# Set gateway and DNS
pihole -a setdhcprouter 192.168.1.1
pihole -a setdhcpdomain local

8 Troubleshooting

a) DNS Resolution Issues

If devices can't resolve DNS:

# Check Pi-hole status
pihole status

# Test DNS resolution
dig @192.168.1.10 example.com

# Check logs for errors
tail -f /var/log/pihole.log

b) Web Interface Not Accessible

If the admin interface is unavailable:

# Check lighttpd status
sudo systemctl status lighttpd

# Restart web service
sudo systemctl restart lighttpd

# Check for port conflicts
sudo netstat -tulpn | grep :80
sudo netstat -tulpn | grep :443

c) Performance Issues

If Pi-hole is slow or unresponsive:

# Check system resources
htop

# Check query log size
ls -lh /etc/pihole/pihole-FTL.db

# Restart Pi-hole services
pihole restartdns

d) Common Error Solutions

Error Solution
FTL failed to start Run pihole reconfigure and check port 53 isn't used by other services
DNS resolution slow Change upstream DNS providers or enable query caching
Web interface password not working Reset with pihole -a -p
Blocking not working Update gravity database with pihole -g
