Zimbra Mail Server Installation for Ubuntu and RHEL

Zimbra is a popular open-source email server and collaboration suite. This guide will walk you through installing Zimbra on both Ubuntu and RHEL-based systems.

Prerequisites

  • Ubuntu 20.04/22.04 LTS or RHEL 8/Rocky Linux 8/AlmaLinux 8
  • Minimum 4GB RAM (8GB recommended for production)
  • At least 20GB of free disk space
  • Static IP address configured
  • Fully Qualified Domain Name (FQDN) pointing to your server
  • Root or sudo privileges
Ubuntu
RHEL/CentOS/Rocky

1 System Preparation (Ubuntu)

Update your system and install required dependencies:

sudo apt update && sudo apt upgrade -y
sudo apt install dnsutils net-tools sudo curl wget unzip -y

Set the hostname and update /etc/hosts:

sudo hostnamectl set-hostname mail.yourdomain.com
echo "127.0.0.1 mail.yourdomain.com" | sudo tee -a /etc/hosts
echo " mail.yourdomain.com" | sudo tee -a /etc/hosts
Note: Replace "mail.yourdomain.com" with your actual domain and with your server's IP address.

2 Download Zimbra (Ubuntu)

Download the latest Zimbra Open Source Edition:

wget https://files.zimbra.com/downloads/10.0.0_GA/zimbra-10.0.0_GA_4604.UBUNTU20_64.20230331160356.tgz

Extract the downloaded file:

tar -xzf zimbra-10.0.0_GA_4604.UBUNTU20_64.20230331160356.tgz
Note: Check the Zimbra downloads page for the latest version.

3 Install Zimbra (Ubuntu)

Navigate to the extracted directory and run the installer:

cd zcs-10.0.0_GA_4604.UBUNTU20_64.20230331160356
sudo ./install.sh

The installer will check system prerequisites. Press Y to accept the license agreement.

When prompted to select packages to install, choose:

3 - Yes (to apply the configuration)

Wait for the installation to complete. This may take several minutes.

1 System Preparation (RHEL/Rocky/AlmaLinux)

Update your system and install required dependencies:

sudo dnf update -y
sudo dnf install epel-release -y
sudo dnf install dnsutils net-tools sudo curl wget unzip perl -y

Set the hostname and update /etc/hosts:

sudo hostnamectl set-hostname mail.yourdomain.com
echo "127.0.0.1 mail.yourdomain.com" | sudo tee -a /etc/hosts
echo " mail.yourdomain.com" | sudo tee -a /etc/hosts

Disable SELinux (required for Zimbra):

sudo setenforce 0
sudo sed -i 's/^SELINUX=.*/SELINUX=permissive/' /etc/selinux/config

2 Download Zimbra (RHEL/Rocky/AlmaLinux)

Download the latest Zimbra Open Source Edition for RHEL 8:

wget https://files.zimbra.com/downloads/10.0.0_GA/zimbra-10.0.0_GA_4604.RHEL8_64.20230331160356.tgz

Extract the downloaded file:

tar -xzf zimbra-10.0.0_GA_4604.RHEL8_64.20230331160356.tgz
Note: Check the Zimbra downloads page for the latest version.

3 Install Zimbra (RHEL/Rocky/AlmaLinux)

Navigate to the extracted directory and run the installer:

cd zcs-10.0.0_GA_4604.RHEL8_64.20230331160356
sudo ./install.sh

The installer will check system prerequisites. Press Y to accept the license agreement.

When prompted to select packages to install, choose:

3 - Yes (to apply the configuration)

Wait for the installation to complete. This may take several minutes.

4 Configure Zimbra

After installation, you'll need to configure Zimbra. Access the admin console:

https://your_server_ip:7071

Log in with the admin password you set during installation.

Configure DNS settings for your domain in the admin console:

1. Go to Configure -> DNS
2. Add your domain
3. Set MX record to point to your Zimbra server
4. Configure SPF, DKIM, and DMARC records for email security

5 Post-Installation Configuration

Enable and start Zimbra services:

sudo su - zimbra
zmcontrol start
zmcontrol status

Set up SSL certificate for secure connections:

zmcertmgr createca -new
zmcertmgr createcrt -new -subject "CN=mail.yourdomain.com"
zmcertmgr deploycrt self
zmcertmgr deployca

Restart Zimbra services to apply changes:

zmcontrol restart

6 Create Mailboxes and Test

Create your first user mailbox:

zmprov ca user1@yourdomain.com password123
zmprov ma user1@yourdomain.com zimbraMailStatus enabled

Access the web client to test email functionality:

https://your_server_ip

Log in with the user credentials you just created.

7 Security Hardening

Configure firewall rules for Zimbra:

# For Ubuntu
sudo ufw allow 25/tcp    # SMTP
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 465/tcp   # SMTP SSL
sudo ufw allow 587/tcp   # SMTP Submission
sudo ufw allow 993/tcp   # IMAP SSL
sudo ufw allow 995/tcp   # POP3 SSL
sudo ufw allow 7071/tcp  # Admin console

# For RHEL/Rocky/AlmaLinux
sudo firewall-cmd --permanent --add-port=25/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=465/tcp
sudo firewall-cmd --permanent --add-port=587/tcp
sudo firewall-cmd --permanent --add-port=993/tcp
sudo firewall-cmd --permanent --add-port=995/tcp
sudo firewall-cmd --permanent --add-port=7071/tcp
sudo firewall-cmd --reload

Enable antivirus and antispam protection:

zmamavisdctl enable
zmantispamctl enable

8 Backup and Maintenance

Set up regular backups:

# Create backup script
echo '#!/bin/bash
/opt/zimbra/bin/zmbackup -f -a all' | sudo tee /opt/zimbra/backup/backup.sh
sudo chmod +x /opt/zimbra/backup/backup.sh

Schedule daily backups with cron:

sudo crontab -e
# Add the following line (runs daily at 2 AM)
0 2 * * * /opt/zimbra/backup/backup.sh

Monitor Zimbra logs:

tail -f /opt/zimbra/log/mailbox.log
tail -f /opt/zimbra/log/nginx.log

9 Troubleshooting

Common issues and solutions:

a) Port already in use

If ports are already in use, check what's running:

sudo netstat -tulpn | grep :25
sudo netstat -tulpn | grep :80

Stop conflicting services (like postfix, sendmail, apache, nginx).

b) DNS resolution issues

Ensure your hostname resolves correctly:

nslookup mail.yourdomain.com
dig mail.yourdomain.com

c) SSL certificate problems

Regenerate and redeploy certificates:

sudo su - zimbra
zmcertmgr createca -new
zmcertmgr createcrt -new -subject "CN=mail.yourdomain.com"
zmcertmgr deploycrt self
zmcertmgr deployca
zmcontrol restart
