How to Install and Secure phpMyAdmin on Ubuntu 20.04 LTS

phpMyAdmin is a free and open-source administration tool for MySQL and MariaDB. This step-by-step guide will walk you through installing and securing phpMyAdmin on Ubuntu 20.04 LTS.

Prerequisites

  • Ubuntu Server 20.04 LTS
  • Root or sudo privileges
  • LAMP stack installed (Linux, Apache, MySQL/MariaDB, PHP)
  • Basic knowledge of Linux command line

1 Install phpMyAdmin

Update your package list and install phpMyAdmin from the Ubuntu repositories:

sudo apt update
sudo apt install phpmyadmin php-mbstring php-zip php-gd php-json php-curl

During the installation, you'll be prompted to choose the web server that should be automatically configured. Select Apache using the space bar and press Enter.

Note: If you're using Nginx instead of Apache, you'll need to manually configure it after the installation.

Next, you'll be asked if you want to configure the database for phpMyAdmin with dbconfig-common. Select Yes and provide the MySQL/MariaDB administrator password when prompted.

2 Enable PHP Mbstring Extension

phpMyAdmin requires the PHP mbstring extension. Enable it with the following command:

sudo phpenmod mbstring

Restart Apache for the changes to take effect:

sudo systemctl restart apache2

3 Test phpMyAdmin Installation

Open your web browser and navigate to your server's domain or IP address followed by /phpmyadmin:

http://your_server_ip/phpmyadmin

You should see the phpMyAdmin login page. Log in with your MySQL/MariaDB root username and password.

Warning: At this point, your phpMyAdmin installation is not secure. Continue with the following steps to enhance its security.

4 Secure phpMyAdmin with Additional Authentication

Add an extra layer of security by setting up HTTP authentication. First, create a password file for authentication:

sudo sh -c 'echo -n "admin:" >> /etc/phpmyadmin/.htpasswd'

Next, add a password for the user "admin" (you can choose any username):

sudo sh -c 'openssl passwd -apr1 >> /etc/phpmyadmin/.htpasswd'

You'll be prompted to enter and verify a password. Remember this password as you'll need it to access phpMyAdmin.

5 Configure Apache for Additional Authentication

Edit the Apache configuration file for phpMyAdmin:

sudo nano /etc/apache2/conf-available/phpmyadmin.conf

Add the following configuration inside the section:

AuthType Basic
AuthName "Restricted Files"
AuthUserFile /etc/phpmyadmin/.htpasswd
Require valid-user

Save and close the file, then restart Apache:

sudo systemctl restart apache2

6 Change the phpMyAdmin URL

To make it harder for automated bots to find your phpMyAdmin installation, change its URL from /phpmyadmin to something unique:

sudo nano /etc/apache2/conf-available/phpmyadmin.conf

Find the line that says Alias /phpmyadmin /usr/share/phpmyadmin and change it to something like:

Alias /mysecretadmin /usr/share/phpmyadmin

Save the file and restart Apache:

sudo systemctl restart apache2

Now you can access phpMyAdmin at your new secret URL:

http://your_server_ip/mysecretadmin

7 Set Up SSL Encryption

If you haven't already, secure your phpMyAdmin installation with SSL encryption. Install Certbot and the Apache plugin:

sudo apt install certbot python3-certbot-apache -y

Obtain an SSL certificate for your domain:

sudo certbot --apache -d your_domain.com

Follow the prompts to complete the SSL certificate installation. Certbot will automatically configure Apache to use HTTPS.

8 Additional Security Measures

For enhanced security, consider implementing these additional measures:

a) Restrict Access by IP Address

Edit the Apache configuration for phpMyAdmin:

sudo nano /etc/apache2/conf-available/phpmyadmin.conf

Add the following inside the section (replace YOUR_IP with your actual IP):

Require ip YOUR_IP
Require all denied

b) Set Up Two-Factor Authentication

phpMyAdmin supports two-factor authentication. To enable it:

sudo apt install libpam-google-authenticator -y

Follow the instructions to set up Google Authenticator on your server.

c) Regularly Update phpMyAdmin

Keep phpMyAdmin updated to protect against known vulnerabilities:

sudo apt update
sudo apt upgrade phpmyadmin

9 Troubleshooting Common Issues

a) "The mbstring extension is missing" Error

If you see this error, make sure the mbstring extension is enabled:

sudo phpenmod mbstring
sudo systemctl restart apache2

b) "404 Not Found" Error

If you get a 404 error when accessing phpMyAdmin, check if the alias is correctly configured:

sudo apache2ctl -S | grep phpmyadmin

c) "Access Denied" for MySQL User

If you can't log in to phpMyAdmin, check your MySQL user privileges:

sudo mysql -u root -p
GRANT ALL PRIVILEGES ON *.* TO 'phpmyadmin'@'localhost' IDENTIFIED BY 'password';
FLUSH PRIVILEGES;
EXIT;
